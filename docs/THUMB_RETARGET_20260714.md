## 1. UDP 패킷 프로토콜 (v3, 현행)

`vision_node_dg5f.py` → `Dg5fReceiver.cs`, UDP 127.0.0.1:5006, float32 little-endian.

| 버전 | 포맷 | 내용 |
|---|---|---|
| v1 | `<20f` (80B) | [0..19] DG5F 관절각[deg] — [0..3]엄지 1_1~1_4 / [4..7]검지 / [8..11]중지 / [12..15]약지 / [16..19]새끼 |
| v2 | `<24f` (96B) | + [20..22] 엄지끝 정규화좌표(해부학 ex,ey,ez) + [23] 핀치 플래그(0/1, 히스테리시스 0.30/0.42) |
| v3 | `<25f` (100B) | + [24] 엄지-검지 끝거리 비율(손길이 정규화, One Euro 필터, 연속값) |

수신부는 **패킷 길이로 버전 판별** — 상위 필드 없으면 미제공 처리(하위호환 양방향).
`GetAngles()` / `GetThumbTip()` / `GetPinchDistance()`로 소비.

## 2. 엄지 구동 아키텍처 (Dg5fThumbIK.cs)

IK 활성 시 `Dg5fHandDriver`는 엄지 채널 주입을 건너뛰고, 매 FixedUpdate 아래 순서로 처리:

```
① 목표 계산: 해부학 목표(tip 복원) ↔ 스냅 목표(로봇 검지끝+1.2cm)를
   핀치 거리비 w로 연속 블렌딩 (pinchNear 0.32 ~ pinchFar 0.55, SmoothStep)
② 목표 스무딩 (targetLerp 10/s)
③ 오차비례 스텝 상한: stepCap = maxStepDeg × clamp01(err/softZone)
④ 정체 감지 감쇠: stallTime(0.6s) 내 최선오차 1mm 미개선 → ×0.95/틱,
   개선 시 ×1.15 점진회복, 목표 3mm 이동 시 전체 재가동 (하한 0.02)
⑤ 순차 CCD (1_1→1_4): 단, 1_1 스텝은 ×_pinchW (평시 0 = ff 전담)
⑥ 자세 prior: 패킷 0..3을 관절별 가중(priorWeights {0,1,1,1})으로 당김.
   틱당 이동 ≤ priorStepRatio(0.3)×stepCap, 핀치 가중만큼 ×(1-w) 약화
⑦ CMC 피드포워드: 채널0 → 1_1 직결 lerp(cmcFeedforward 8/s), ×(1-w)
```

설계 원칙 (이번 세션에서 실측으로 확립):
- **같은 관절을 두 컨트롤러가 다투면 리밋사이클** — prior는 stepCap의 30%로 캡,
  1_1은 평시 CCD 제외(ff 전담)·핀치 시 역할 교대.
- **접촉 > 자세**: 핀치 가중이 오르면 prior/ff 모두 비례 소멸, CCD가 전권.
- **정지 판정은 시간창** — 틱당 개선 판정은 물리 추종 지연을 정체로 오판.

## 3. 시행착오 기록 (기각/수정 이력)

1. **prior 초기 구현(무제한 kP)**: 핀치 중 prior가 접촉 목표를 반대로 당겨 3.8cm 정체
   → ×(1-핀치가중) 약화로 해결.
2. **"떨림 = prior 탓" 오진**: prior 스텝캡을 넣어도 진동 불변(p2p 17~23°) →
   prior OFF A/B에서도 15~22° 동일 = **prior 무죄, CCD 자체가 범인**.
   실측 근거: 정지 입력에 1_1/1_3/1_4만 요동(1_2 안정), 패킷 끊기면 완전 정지
   (물리 결백), |tgt-act|max = 15.000° 정확(와인드업 가드 포화).
3. **stall damper 1차(틱당 0.2mm 개선 판정)**: 리밋사이클은 잡았으나 물리 추종
   지연까지 정체로 오판 → 핀치 수렴이 2.4cm에서 정체. 시간창 판정으로 교체.
4. **stall damper 2차(개선 시 권한 즉시 1.0 리셋)**: 감쇠된 사이클이 재점화
   (15~19° 재발) → ×1.15 점진회복으로 교체.
5. **CMC 옛 프록시(손목-CMC-MCP 3점각)**: 세 점 준일직선 → 조건수 불량 →
   §20-2에서 게이트. 신프록시 = 중수골(CMC→MCP)의 손바닥 평면 이탈각 arcsin —
   전 구간 안정. **옛 보정값(0.376~0.702)은 신프록시와 무관 → json에서 제거**.

## 4. 검증 데이터 (전부 Dg5fJointLogger 50Hz CSV, 5초 구간별 p2p)

- **정지 프로브(수정 전)**: tgt_1_1 p2p 17.3°/act 23.4° 지속, 50초 내내.
- **정지 프로브(최종)**: 65°(초기 과도)→7.3→5.9→1.2→**0.7°** 단조 감쇠, 재점화 0.
- **핀치**: 검지 62/52/38° 조건에서 엄지-검지 끝거리 1.2cm(=pinchOffset) 정확 수렴
  — §20-3 oktip 벤치마크 동등. (검지 컬 얕으면 작업공간 밖 — 기존 제약 동일)
- **CMC 스윕(ch0 삼각파 0→-60→0, 왼손)**: rx↔act 1_1 상관 0.991, |오차| 평균 1.68°,
  정지 hold 시 1_1 p2p 0.07°.
- **잔여 한계**: 모순 입력(1_1 강제 + 손끝 목표 고정) 최악 조건에서 1_3/1_4가
  1~2.5°/s 미세 숨쉬기 = 감쇠 하한(0.02) 크롤링. 실손 입력은 신호가 일관돼 덜함.

## 5. 튜닝 노브 (Dg5fThumbIK 인스펙터)

| 필드 | 기본 | 올리면 | 내리면 |
|---|---|---|---|
| targetLerp | 10/s | 반응↑ 노이즈 통과↑ | 부드러움↑ 지연↑ |
| softZone | 1.5cm | 접근 감속 구간↑ | 종단 속도↑ 오버슈트 위험 |
| stallTime | 0.6s | 정체 판정 관대 | 까딱임 빨리 죽음, 느린 수렴 오판 위험 |
| priorGain / priorStepRatio | 3/s / 0.3 | 사람 모양 재현↑ | 위치 정확도 우선 |
| cmcFeedforward | 8/s | 앞뒤 반응↑ | 노이즈 억제 |
| pinchNear/Far | 0.32/0.55 | — | 스냅 개시 거리 조절 |

수렴이 느리다는 체감이 남으면: stallTime 0.6→0.4, 감쇠율(코드) 0.95→0.90,
하한 0.02→0.08, 재가동 임계 3mm→1.5mm 순으로 실험.

## 6. 라이브 확인 체크리스트 (사용자)

1. `python vision_node_dg5f.py left` — `[send]` 첫 값(ch0)이 엄지를 손바닥
   앞쪽으로 움직일 때 **증가**하는지. 반대면 dg5f_angles.py 채널 테이블에서
   thumb_cmc (dg_min,dg_max)=(65.0,0.0) 스왑.
2. `calibrate_dg5f.py` 재실행 — thumb_cmc 신프록시 범위 학습(옛 항목은 제거됨).
3. 엄지 떨림/앞뒤 재현/OK 접촉 체감 → 문제 시 `Dg5fJointLogger` +
   `analyze_teleop.py latest --channels thumb`으로 목표 vs 추종 분리 진단.
