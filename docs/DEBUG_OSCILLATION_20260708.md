## 1. 실험 A — 정상 vs 진동 관절 기하 전수 비교 (기각)

**가설**: 병든 관절(p, l, o, k)은 xDrive twist축(anchorRotation 로컬 X)과 실제 회전축이 어긋나 있고, 정상 관절(z, virtual_i)은 일치할 것이다.
**결과**: **기각.** 앵커·축·스케일·URDF 대조 전 항목에서 6개 관절이 동일하게 결백. 갈리는 건 "자기 축을 중심으로 얼마나 폭주하는가"뿐.

### 1-1. 비교표 (에디터 실측 + URDF 원본 대조)

| 항목 | z (정상) | virtual_i (정상) | l | p | k | o |
|---|---|---|---|---|---|---|
| matchAnchors | True | True | True | True | True | True |

| parentAnchorPos | (-0.0263,0,-0.0169) | (0.006,0.056,0.0254) | (0,0,0) | (0,0,0.0480) | (0,0,~0) | (0,0,0.0500) |
| 앵커 월드 불일치 | 0.0001mm/0.000° | 0.0001mm/0.000° | 0.0001mm/0.000° | 0.0002mm/0.000° | 0.0002mm/0.000° | 0.0002mm/0.000° |

### 1-2. 핵심 교훈 두 가지

1. **월드 angularVelocity로 축 정렬을 재면 안 된다.** 정상 관절이 오히려 59~71° 어긋나 보였는데, 실체는 자기 관절은 거의 정지(잔여 p2p 0.12°)인 채 **손바닥 흔들림을 물려받은 것**. 반드시 부모-상대 ω(ω_child − ω_parent)로 잴 것 → 그러면 6개 전부 0.2~1.9°.
2. **PhysX articulation(reduced coordinate)은 구조적으로 축 이탈이 불가능**하다. 축/앵커 어긋남 계열 가설은 앞으로 세울 필요 없음.

부수 실측: e1(손바닥, 팔에 fixed)이 wmag 최대 3.2 rad/s로 흔들림 = 진동 에너지가 팔 체인으로 역류. e2(mimic j5)도 56° 스윙. URDF origin xyz는 URDF→Unity 좌표변환 (x,y,z)→(−y,z,x) 규칙대로 전부 일치, z의 axis 부호(−1)는 anchorRotation z=90°(나머지 270°)로 정확 반영.

## 2. 실험 B — Solver Type: Projected → Temporal Gauss Seidel (부분 개선)

**배경**: 공식 URDF Importer 튜토리얼이 관절 불규칙 거동 방지를 위해 TGS를 명시 요구하는데 한 번도 확인 안 했었음. 확인 결과 **PGS(기본값)** 상태였음.
**방법**: `ProjectSettings/DynamicsManager.asset`의 `m_SolverType` 0→1만 변경(다른 설정 일절 불변), 각각 **플레이 새로 시작** 후 동일 프로브(주먹↔펴기 2s×4, UDP 30Hz) 12초 로깅.

### 2-2. 판정

- **명확한 개선**: 전체 진폭 대략 절반(p 405→192, l 144→61), 주먹 추종 오차도 절반 수준, l·k의 리밋 침범 깊이 −27/−36 → −6/−10°.
- **그러나 미해결**: neg 침범 빈도는 그대로(22~43%), p·o 원위지골은 여전히 리밋 밖 −72~−86°까지 관통, p는 정상상태 p2p가 오히려 악화(77→107). **TGS는 필요조건이지 근본 수정이 아님** — §8 원인(초경량 mimic 지골 드라이브)은 그대로 유효.
- **현재 상태: TGS 유지 중**(임포터 공식 요구 설정이기도 함). 되돌리려면 `m_SolverType=0`.

## 3. 실험 C — 임포터 기본값 관성 링크 수정 (✅ 진동 완전 해결)

**배경 — 질량/관성 전수 측정(수정 없이 측정만)에서 발견한 4가지**:

1. **팁 링크 5개(thtip/fftip/mftip/rftip/lftip)가 임포터 기본값 mass=1.0kg·관성(1,1,1)kg·m² 그대로** —
   0.02kg mimic 원위지골 아래 fixed joint로 매달려 체인 끝에서 질량 ×50, 관성 ×1.7e6 역전.
2. virtual 너클 4개 중 **virtual_k만 7/2 수동값(1e-5) 잔존, virtual_l/j/i는 (1,1,1) 기본값** —
   콜라이더가 없어 autoTensor=True여도 자동산출 불가, 기본값이 그대로 남는 구조.
3. 손 링크 관성은 전부 autoTensor 자동산출 — URDF inertial이 있는 7개(c,p,t,s,r,m,q)조차
   질량만 URDF 일치, 관성값은 미사용(7/2 ResetInertiaTensor 흔적).
4. 팔↔손 더미 flange/tool0/svh_base_link도 1kg/(1,1,1).

검지 체인 관성 캐스케이드: `e1 1.6e-4 → virtual_l 1.0(×6250) → l 2.3e-6(÷43만) → t 5.9e-7 → fftip 1.0(×1.7e6)`
— 한 체인에서 6자릿수를 두 번 오르내림. §8의 "t+fftip 제거 → 완전 안정"과 정확히 맞물림.

**수정(기본값 링크 11개만, 그 외 드라이브·솔버 등 일절 불변)**:

| 대상 | mass | inertiaTensor | 근거 |
|---|---|---|---|
| 팁 5개 (thtip/fftip/mftip/rftip/lftip) | 0.005 | (1e-6)³ | 무기하 프레임 → 최소 유한값 |
| virtual_l/j/i | 0.01 유지 | (1e-5)³ | virtual_k와 통일 |
| flange/tool0/svh_base_link | 0.1 | (1e-4)³ | 더미 플랜지 수준 |

씬 백업: `backups/SampleScene_before_defaultinertia_102827.unity`. 수정 후 씬 저장됨.

정상상태(목표 불변 구간, 진입 0.5s 제외) 잔여 진동: **6채널 전부 p2p 0.00° / max|err| 0.01°**.
물리 동결 아님(명령 전환 시 76.4° 이동을 정상 추적). 콘솔 에러 없음.

**결론**: §8에서 특정한 mimic 원위지골 드라이브는 *트리거*였고, 실제 에너지원은
**임포터 기본값 1kg/(1,1,1) 관성의 팁·virtual·더미 링크들**. 이들이 잡히자 드라이브
10000/200 그대로도 완벽 수렴 → §8 수정 후보(mimic 게인 하향 등)는 **불필요해져 폐기**.
데이터: `after_fixdefaults.csv` (기준 `after_tgs.csv`).
