# DG5F 보정 가이드 — calibrate_dg5f.py 실행 시 해야 할 동작

보정은 채널별로 "내 손이 실제로 움직이는 각도 범위(min/max)"를 기록하는 과정이다.
**그 채널이 진짜 최대로 움직이는 자세를 한 번씩은 만들어줘야** 로봇 손 매핑이 정확해진다.
빠뜨린 채널은 기본값이 남아 해당 손가락이 덜 굽거나 과하게 굽는다.

```bash
cd C:\Users\dltmd\Desktop\KDT\dg5f
python calibrate_dg5f.py     # 종료: 웹캠 창에서 q → dg5f_calibration.json 자동 저장
```

## 촬영 조건 (동작만큼 중요 — 먼저 읽을 것)

| 조건 | 이유 |
|---|---|
| **손목 곧게 고정, 손가락만 움직이기** | 첫 마디(MCP) 굽힘각을 손목 랜드마크 기준으로 계산 — 손목을 꺾으면 손가락 안 굽혀도 굽힌 걸로 오염됨. **가장 흔한 오염원** |
| **손바닥이 카메라 정면** | 단일 카메라라 깊이(z) 부정확 → 손이 옆으로 돌면 각도 왜곡. 실제 텔레옵 때 쓸 방향으로 보정 |
| **극값에서 1~2초 멈추기** | 스치듯 지나가면 안정 관측이 안 됨. 펴기 끝·주먹 끝에서 잠깐 유지 |
| **천천히, 흔들지 않기** | min/max 누적 방식 — 모션블러 튐값이 극값을 오염. 망친 것 같으면 q로 끝내고 재실행(덮어쓰기) |
| 손은 화면 중앙, 프레임의 1/2~2/3 크기 | 너무 작으면 랜드마크 정밀도↓, 너무 크면 잘림 |
| 텔레옵에 쓸 손으로 보정 | 보정 자체는 좌우 공용(raw 각도가 해부학 기준). 현재 씬은 **왼손 결합 모델(ur5e_dg5f_left)** — vision_node 실행 시 `left` 인자 |

## 보정 동작 체크리스트 (각 3회, 극값에서 1~2초 유지)

1. ☐ **펴기 + 뒤로 살짝 젖히기(과신전) ↔ 주먹 꽉 쥐기**
   — 굽힘 채널 전체의 기본 범위. 과신전을 해야 "완전히 편 상태"가 진짜 min으로 잡힘
   (안 하면 로봇 손이 항상 살짝 굽은 채로 시작함)
2. ☐ **손가락 하나씩 개별로 굽혔다 펴기** (검지→중지→약지→새끼→엄지)
   — 채널 분리 + 개별 극값
3. ☐ **갈고리(hook) 자세**: 첫 마디는 편 채 둘째·셋째 마디만 구부리기 (손톱이 손바닥 향하게)
   — PIP/DIP 채널의 진짜 최대 분리 (주먹만으로는 안 나옴)
4. ☐ **ㄱ자(테이블탑) 자세**: 첫 마디만 90° 꺾고 둘째·셋째 마디는 쭉 펴기
   — MCP 채널의 진짜 최대 분리
5. ☐ **엄지: 쫙 벌리기 ↔ 엄지 끝으로 새끼손가락 뿌리 터치**
   — 엄지 대향(1_2)의 최대는 "접기"가 아니라 손바닥을 가로지르는 이 동작
6. ☐ **엄지-검지 OK 사인 (집기)**
   — 엄지 MCP/IP 실사용 범위 (파지 핵심 자세)
7. ☐ **손가락 편 채 쫙 벌리기 ↔ 모으기**
   — 벌림(n_1) 채널 (지금은 게이트로 꺼져 있지만 해제 대비 미리 보정)
8. ☐ **손 오므려 컵 모양** (새끼가 엄지 쪽으로 접히게)
   — 새끼 손바닥접기(5_1) 채널 (게이트 해제 대비)
9. ☐ **엄지 앞뒤(안테포지션)**: 손바닥 정면 유지한 채 엄지만 카메라 쪽으로 들었다
   ↔ 손바닥 평면으로 되돌리기
   — 엄지 CMC(1_1, thumb_cmc) 채널. **2026-07-14 신프록시 전환으로 옛 보정값이
   삭제돼 있어 이번 보정에서 반드시 필요** (빠뜨리면 엄지 앞뒤가 기본값 매핑)
10. ☐ **엄지 최대한 쭉 뻗기 (2026-07-15 v2 신규 — 필수)**: 엄지를 손에서 최대한
    멀리 곧게 뻗은 상태를 **방향 2~3가지**(옆으로/대각선/벌린 채)로 각 **2초 유지**
    — 엄지 손끝 IK의 스케일 기준 `thumb_reach_ratio`(|엄지끝−CMC| 직선/손길이의
    p95)가 여기서 잡힘. **손바닥 평면 안에서(화면에 보이게) 뻗는 방향을 꼭 포함**
    (카메라 쪽으로만 뻗으면 z 압축으로 짧게 측정됨). 이 동작을 빠뜨리면 비율이
    작게 저장돼 라이브에서 엄지가 덜 뻗는다.

전체 약 1분 20초. 순서는 위 번호 순서 추천.

## 종료 후 확인

- 콘솔에 채널별 `human_min ~ human_max` 출력 + `dg5f_calibration.json` 저장됨(v2 형식).
- **정상 눈대중**: 굽힘 채널(mcp/pip/dip)은 대략 min 0.0~0.3 / max 1.0~2.0 (rad).
  범위 폭이 0.3 미만으로 좁으면 그 채널 동작이 빠진 것 → 재실행.
- **`thumb_reach_ratio = 0.xx` 줄 확인 (2026-07-15 v2 — 구 `thumb_chain_ratio` 대체)**:
  엄지 손끝 IK의 스케일 기준(|엄지끝−CMC| 직선/손길이 p95). **정상 범위 대략 0.9~1.2**.
  콘솔에 "0.85 미만 — 엄지를 충분히 안 편 것 같음" 경고가 뜨면 동작 10번을 제대로
  하고 재보정.
- 다음 실행부터 `vision_node_dg5f.py`가 시작 시 `보정값 로드됨` 메시지와 함께 자동 적용.
  `엄지 최대도달 비율 보정값 사용: 0.xx` 로그도 같이 나오는지 확인.
  **⚠️ "보정 파일이 구버전(v1…)" 경고가 나오면** 재보정 전 상태로 돌고 있는 것 —
  이 가이드대로 calibrate_dg5f.py를 다시 실행해야 함 (v1의 마디합 값은 의미가 달라
  자동 폐기됨, 각도 범위는 그대로 사용됨).

## 문제 있을 때

- 특정 손가락이 덜/과하게 굽음 → 콘솔의 관측 결과에서 해당 채널 범위 확인, 해당 동작 다시 하고 재보정.
- 엄지가 반대로 움직임 → 보정 문제가 아니라 방향 문제 — `dg5f_angles.py`의 해당 채널
  (dg_min, dg_max) 스왑 (개발 담당에게 로그 파일명과 함께 전달).
- 재보정은 언제든 `python calibrate_dg5f.py` 재실행 (이전 값 덮어씀).
